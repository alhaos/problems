# Алгоритм «Хак Госпера»

## Реализация на Go

```go

func NextCombination(x uint8) uint8 {
	c := x & -x             // шаг 1: младший установленный бит
	r := x + c              // шаг 2: сдвигаем правый блок единиц влево
	changed := r ^ x        // шаг 3: биты которые изменились
	shifted := changed >> 2 // шаг 4: убираем два лишних бита
	tail := shifted / c     // шаг 5: нормализуем к нулевой позиции
	next := tail | r        // шаг 6: собираем результат
	return next
}

```

## Разбор на примере `x = 6`

### Шаг 1: Младший установленный бит

Необходимо установить младший из установленных бит, для этого совершаем
логическое сложение числа с его отрицательным значением.

```go

c := x & -x

```

```text

  00000110  (6)
& 11111010  (-6)
  --------
  00000010  (2)

```

### Шаг 2: Сдвигаем правый блок единиц влево

Это главное действие алгоритма. Прибавление c к x вызывает цепочку переносов
через весь правый блок единиц — они все гасятся и появляется одна новая единица
левее блока. Так мы «продвигаем» комбинацию вперёд.

```go

r := x + c

```

```text

  00000110  (6)
+ 00000010  (2)
----------
  00001000  (2)

```

### Шаг 3: Биты которые изменились

Отслеживаем что поменялось

```go

changed := r ^ x

```

```text

  00001000  (8)
^ 00000110  (6)
----------
  00001110  (14)

```

### Шаг 4: Убираем два лишних бита

В результате XOR есть два «служебных» бита которые нас не интересуют: новая
единица в r и позиция самого c. Сдвиг на 2 убирает их, оставляя только счётчик
потерянных единиц.

```go

shifted := changed >> 2

```

```text

  00001110  (14)
>>2
----------
  00000011  (3)

```

### Шаг 5: Нормализуем к нулевой позиции

После сдвига на 2 потерянные единицы ещё находятся не на своём месте — они
смещены на k позиций вправо где k — позиция бита c. Деление на c = 2^k это
просто правый сдвиг на k, который опускает их к нулевому биту. Теперь хвост
готов к вставке.

```go

tail := shifted / c

```

```text

  00000011  (3)
/ 2
----------
  00000001  (1)

```

## Шаг 6: Собираем результат

Объединяем два независимых куска: r содержит продвинутую левую часть, tail
содержит восстановленный правый хвост. OR склеивает их в финальное число с тем
же количеством единиц.

```go

next := tail | r

```

```text

  00001000  (8)
| 00000001  (1)
----------
  00001001  (9)

```
