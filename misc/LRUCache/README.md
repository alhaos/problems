# LRU Cache

Реализация структуры данных **LRU Cache** (Least Recently Used) — кэш с фиксированной ёмкостью, который вытесняет давно неиспользуемый элемент при переполнении.

## Интерфейс

```go
Constructor(capacity int) LRUCache  // создать кэш с заданной ёмкостью
Get(key int) int                    // вернуть значение по ключу или -1
Put(key int, value int)             // добавить/обновить пару ключ-значение
```

## Пример

```go
cache := Constructor(2)

cache.Put(1, 1)   // кэш: {1=1}
cache.Put(2, 2)   // кэш: {1=1, 2=2}
cache.Get(1)      // → 1       кэш: {2=2, 1=1}  (1 стал свежим)
cache.Put(3, 3)   // кэш полон → вытесняем 2
                  // кэш: {1=1, 3=3}
cache.Get(2)      // → -1      (2 вытеснен)
cache.Get(3)      // → 3
```

## Ограничения

- `1 <= capacity <= 3000`
- `0 <= key, value <= 10^4`
- Операций не более `2 * 10^5`
- `Get` и `Put` должны работать за **O(1)**
