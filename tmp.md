# Порядок установки обновления ТМ

## 1. Действия до выполнения обновления

На время выполнения обновления необходимо:
Сервис Загрузчика ТМ остановить (кроме Сервиса по схеме Предикс predix_tm).
Конфигуратор Загрузчика ТМ закрыть.
Джобы по Загрузчику ТМ остановить.

## 2. Выполнение обновления

Обновление по ТП и выравниванию версий для всех схем, где имеется Загрузчик ТМ
(ОДС, Предикс, ...) По скриптам ниже описано, на каких схемах следует выполнять.
Файлы Сервиса и Конфигуратора обновить(заменить) по всем схемам, кроме Предикс
(по схеме predix_tm было отправлено обновление 13782).

В обновлении реализовано следующее:

- Изменены пакеты по удалению и добавлению партиций -установка параметра ожидания получения блокировки dml
- Логирование ошибок соединения
- Актуализация алгоритмов распознавания
- Актуализация справочника параметров wellop_param
- Заявка #TL110009 по исправлению алгоритмов распознавания

Месторождение (любой вид) + Имя скважины (любой числовой суффикс) -> Нефтяные скважины
Месторождение (любой вид) + Имя скважины (любой числовой суффикс) -> Нефтяные
скважины -> Компоновка

Поиск по любому числовому суффиксу, а именно: удаление последних 2х цифр из имени
скважины, поиск скважин по изменённому имени и любым суффиксом, который
начинается не с цифр.

Скрипты находятся в архиве sql_tm_260420.7z. Пароль 2026

2.1. Загрузчик ТМ Предикс (схема predix_tm)

Для схемы Загрузчик ТМ Предикс (схема predix_tm) следует выполнить только
указанные скрипты:

10. tm4_main.sql
11. tm4_recognize.sql

2.2. Загрузчик ТМ ОДС и др. (схема cds, ..)

Для схемы Загрузчик ТМ ОДС (и т.д.) выполнить следующие пункты:

2.2.1. Выполнить скрипты

```text

00.	tm4_rec_skvcod_add.sql
00.02. tm4_unit_add_v3.sql
04. tm4_rec_add_wellwork.sql
07._wellop_param_260324.sql
10. tm4_main.sql
11. tm4_recognize.sql
17. tm4_func_fill.sql
20. tm_clear_tables.sql
23. tm4_add_partitions.sql

```

В скрипте 07._wellop_param_260324.sql все актуальные данные по справочнику.
При выполнении скрипта пропускать дубли, игнорировать.

Папка local_index
Использование локальных индексов в таблицах с партициями.

```text

01.	alter_tm4_data_local.sql
02.	alter_tm4_result_local.sql
03.	alter_tm4_data_index.sql

```

Для джобов создания партиций и очистки установить интервал, равный примерно
1-2 часам (SYSDATE + 1/24), разнести по времени выполнения друг с другом.


Для ОГ, в которых имеется выделенный сервер с Сервисом Загрузчика ТМ и отдельный
сервер для разбора замеров. (Уват, Башнефть-Полюс)

На сервере, где происходит разбор замеров (имеются пакеты tm4_*):

- Скрипты;
- Обновление файлов Конфигуратора.

На выделенном сервере с Сервисом Загрузчика ТМ:

- Обновление файлов Сервиса.

2.2.2. Обновление файлов Загрузчика ТМ
Сохранить предыдущие версии Конфигуратора и Сервиса Загрузчика ТМ.
Обновить файлы Конфигуратора и Сервиса Загрузчика ТМ (скопировать в папки) в
соответствии с битностью из архивов:

```text 

Ngt.Tm4.Configurator.Bin_x*_4.3.0.1789 - Конфигуратор
Ngt.Tm4.Loader.Bin_x*_4.3.0.1789 – Сервис
Пароль 1

```

3. Действия после выполнения обновления

После выполнения обновления необходимо:

- Скомпилировать пакеты tm4_*, tm_clear_tables.
- Запустить джобы Загрузчика ТМ (для джобов создания партиций и очистки учесть интервалы).
- Проверить работу джобов
- Запустить Сервис Загрузчика ТМ (кроме схемы Предикс predix_tm, где не останавливали Сервис).
•	Проверить работу Сервиса – загрузку замеров
•	Открыть Конфигуратор.
•	Проверить разбор замеров в любом сигнале.
