# Worker Pool

## Описание

Worker Pool — паттерн конкурентной обработки задач в Go.
Суть: фиксированное количество горутин-воркеров читают задачи из общего канала
и пишут результаты в общий канал результатов. Количество воркеров задаётся 
заранее — это позволяет контролировать нагрузку и не плодить горутины бесконтрольно.

jobs → [worker1] ─┐
jobs → [worker2] ─┼─► out
jobs → [worker3] ─┘

Где применяется:

ограниченный параллелизм при обращении к БД или внешним API
обработка очереди задач с контролем нагрузки
CPU-интенсивные вычисления — воркеров обычно runtime.NumCPU()
любая ситуация где нужно обработать N задач не более чем M горутинами одновременно

## Задача

Реализовать пул воркеров который конкурентно обрабатывает задачи из общего канала.

## Сигнатура

```go
func WorkerPool(ctx context.Context, numWorkers int, jobs <-chan int) <-chan int
```

## Пример

```go
ctx := context.Background()

jobs := make(chan int, 5)
for _, n := range []int{1, 2, 3, 4, 5} {
    jobs <- n
}
close(jobs)

results := WorkerPool(ctx, 3, jobs)
for r := range results {
    fmt.Println(r) // 1, 4, 9, 16, 25 (порядок не гарантирован)
}
```

## Требования

- Запустить ровно `numWorkers` горутин
- Каждый воркер читает задачи из общего канала `jobs`
- Результат пишется в общий канал `results`
- Все воркеры завершаются когда `jobs` закрыт или контекст отменён
- `results` закрывается только после завершения всех воркеров


## Ключевые моменты

```go
// wg.Wait() в отдельной горутине — иначе дедлок
go func() {
    wg.Wait()
    close(out)
}()

return out // возвращаем сразу, не блокируемся
```
